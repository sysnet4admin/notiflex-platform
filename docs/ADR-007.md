# ADR-007: Canary 배포 전략으로 전환

## 상태

**채택 (Accepted)**

## 맥락 (Context)

- 현재 Blue/Green 배포 전략은 안정적이지만, 신규 버전에 문제가 발생했을 때 전체 트래픽이 일시에 영향을 받는 위험이 있습니다.
- 실제 운영 환경의 일부 트래픽으로 신규 버전의 안정성을 먼저 검증하는 과정이 필요합니다.
- 점진적으로 트래픽을 전환하며 모니터링을 통해 신규 버전의 성능 및 기능 이상 여부를 확인할 수 있는 고도화된 배포 전략이 요구됩니다.

## 결정 (Decision)

**Argo Rollouts의 배포 전략을 Blue/Green에서 Canary로 전환합니다.**

- `setWeight` 단계를 통해 신규 버전(Canary)에 적은 비율의 트래픽(예: 5%)을 먼저 보내고, 정해진 시간 동안 `pause`하여 안정성을 검증합니다.
- Prometheus 메트릭(예: HTTP 에러율, 응답 시간)을 분석하여 자동으로 롤백 여부를 판단하는 AnalysisTemplate을 적용합니다.
- 검증이 완료되면 점진적으로 트래픽 비율을 높여 최종적으로 100%로 전환합니다.

## 이유 (Rationale)

- **위험 감소**: 소수의 사용자에게만 신규 버전을 노출시키므로, 장애 발생 시 전체 서비스에 미치는 영향을 최소화할 수 있습니다.
- **신뢰성 있는 검증**: 실제 운영 트래픽을 통해 신규 버전의 성능과 안정성을 객관적인 지표로 검증할 수 있습니다.
- **자동화된 롤백**: 모니터링 시스템과 연동하여 문제 발생 시 사람의 개입 없이 자동으로 이전 버전으로 롤백하여 서비스 안정성을 확보합니다.
- **유연한 트래픽 제어**: 서비스의 특성과 중요도에 따라 트래픽 전환 비율과 검증 시간을 유연하게 조절할 수 있습니다.
